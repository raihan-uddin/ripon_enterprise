# Architecture Overview

## What This Is

A PHP/Yii 1.x enterprise ERP system for inventory management with integrated sales, procurement, accounting, loan, and user management modules. Runs on Apache + PHP + MySQL with no build pipeline.

## Request Flow

```
index.php (entry point, YII_DEBUG=TRUE)
  → protected/config/main.php (loads modules, DB, auth, routing)
  → protected/config/urlManager.php (path-based route matching)
  → Controller::action() (protected/controllers/ or protected/modules/*/controllers/)
  → Model (protected/models/ or protected/modules/*/models/)
  → themes/erp/views/.../*.php (rendered output via theme)
```

## Configuration

| File | Purpose |
|------|---------|
| `protected/config/main.php` | App bootstrap: modules, components, imports, DB |
| `protected/config/db.php` | MySQL connection (localhost, ripon_ent, root, no password) |
| `protected/config/urlManager.php` | URL routing rules (path format) |
| `protected/config/params.php` | App parameters (company name, timezone: Asia/Dhaka) |

## Frontend Stack

- Bootstrap 3.3.7 + jQuery 3.2.1 (CDN)
- Font Awesome 4.7.0 for icons
- AdminLTE CSS/JS loaded in main layout
- toastr for notifications, moment.js for dates
- Custom CSS in `themes/erp/css/` and root `css/`
- Print styles: `css/print.css`

## Key Patterns

- **All controllers** extend `Controller.php` → `RController` (rights module RBAC)
- **All models** extend Yii's `CActiveRecord`
- **All views** live in `themes/erp/views/` (never in `protected/views/`)
- **DB transactions**: `Yii::app()->db->beginTransaction()` → save → commit/rollback
- **Permissions**: `Yii::app()->user->checkAccess('Permission.Key')`
- **Session state**: business_id and branch_id stored via `Yii::app()->user->setState()`
- **Gii code generator**: available at `/gii` (password: admin)
